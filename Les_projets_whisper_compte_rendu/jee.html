Le Back-end

REST

L'application suit une architecture REST (REpresentational State Transfer), qui est basée sur le protocole HTTP et URI (pour l'adressege des ressources), et les types MIME (pour représenter les ressources). REST permet d'utiliser un protocole de communication sans état (stateless), selon une architecture client-serveur. REST est un style d'architecture pour réaliser des réseaux. La communication entre le client et le serveur passe par du HTTP, ce qui simplifie grandement les échanges, en utilisant des méthodes telles que GET, POST, PUT, DELETE. Ces méthodes permettent de réaliser des opérations CRUD (Create, Read, Update, Delete). Cette architecture est notament utilisée par des entreprises telles que Google, Yahoo, Ebay et Amazon. Comme REST n'est pas un standard, il n'existe donc pas de spécification REST pour construire des applications web.


Jersey

Nous avons utilisé Jersey pour créer une application RESTfull, qui est un framework open source qui étends l'API JAX-RS (implémentation de référence de la JSR 311 et de la JSR 339). JAX-RS est un standard. De plus Jeersey fournit sa propre API pour simplifier le développement des services web RESTfull. Jersey fonctionne optimalement avec avec le serveur d'application JEE GlassFish, c'est pourquoi nous l'avons également utilisé pour ce projet. L'application Whisper retourne du XML et du JSON.


Maven

Nous avons également utilisé le logiciel libre Maven, qui permet de configurer la gestion du projet, via un format XML. Cela permet de téléchager les dépendances nécessaire au projet, d'automatiser les builds, de synchroniser les modules, etc. Ces configurations se trouvent dans le fichier pom.xml.


Tests

Deux types de tests ont étés mis en place, afin de tester les méthodes implémentées. Le premier type de tests sont les tests fonctionnels. Le second type de tests sont les tests d'intégration. Pour cela, différents outils ont été utilisés. C'est notament le cas de JUnit, qui est un framework pour réaliser des tests unitaires. L'extention de JUnit, RESTfuse, a été utilisée afin de mettre en place les tests d'intégration. Cette extention est spécialiste pour le REST.


Le Front-end

AngularJS

La partie client a été mise en place via l'utilisation du framework libre et open source AngularJS. C'est un framework javascript MVC (Modèle-Vue-Contrôleur) développé par Google. AngularJS permet d'adapter et d'étendre le langage HTML pour mettre en place un contenu dynamique poussé. Cela est rendu possible via la mise en place d'un 'data-binding' qui est bi-directionnel. De ce fait, les vues et les modèles synchronisent de façon automatique. La logique métier et les manipulations du DOM sont découplés grâce à AngularJS. Le coté serveur et client sont totalement découplés, afin de permettre un développement parallèles des deux cotés et ainsi permettre une réutilisabilité de chacun des deux cotés. De plus, ce framework facilite les tâches à mettre en place dans le site. Un des gros avantages d'utiliser ce framework, est que cela allège les contraintes liées au back-end et permet de produire des applications plus légères.


Twitter Bootstrap

En ce qui concerne le design du site, nous avons utilisé le kit CSS Twitter Bootstrap. C'est un framework de front-end qui repose sur un système de grilles et contient des templates déja prêtes à l'emploi, permetant de réaliser des sites web très esthétiques.
